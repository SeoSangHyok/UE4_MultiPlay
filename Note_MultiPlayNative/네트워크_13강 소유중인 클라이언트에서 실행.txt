1. 네트워크 테스트시 에디터 개인설정의 [플레이](상단 플레이버튼 옵션을 펼친 - [고급세팅] 으로 들어가면 나옴)항목에서 단일 프로세스 사용을 체크 해지하면 각각 따로 실행되는 클라이언트처럼 테스트가 가능하다.

2. 위처럼 처리하면 각기 다른 프로세스를 사용하기때문에 디버깅시 흐름관찰에 좀더 효과적이니 참고

3. 하지만 데디서버를 띄울수없는 단점도 있다.

4. 네트워크에서 소유중인 클라이언트에서 실행 함수
  - 해당 액터를 가지고 있는 플레이어 컨트롤러의 컴퓨터에서만 실행된다는 의미
    -- 서버에서 해당 함수를 실행하면 해당 액터의 주인 클라이언트에 패킷을 날려 해당 클라이언트에서만 실행
    -- 해당 액터의 주인 클라이언트에서 실행시 바로 실행. 서버로 날리지 않는다는점에 유의!!
    -- 다른 클라이언트에는 무시된다.
  - 네이티브에서 구현은 해당 함수선언시 UFUNCTION 매크로에 Client 를 추가. 전송보장효과를 누리려면 Reliable를 추가하면 된다.( UFUNCTION(Client, Reliable) )
    -- 함수 구현은 함수명_Implementation() 으로 구현
